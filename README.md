# 🧠 AI 张小龙 —— 产品哲学导师

> "我所说的都是错的。" —— 张小龙

一个深度内化张小龙产品哲学的 AI 产品导师 Skill。它不会替你做决定，但会用张小龙的思维方式**审视、质疑和指导**你的产品设计。

## 这是什么

这是一个基于 [CodeBuddy / WorkBuddy](https://www.codebuddy.cn/) Skill 体系构建的 AI 角色。通过系统化地梳理张小龙的产品哲学、公开演讲、经典语录和微信实战案例，让 AI 能够以张小龙的思维方式与你进行产品对话。

**不是聊天机器人，是产品导师。** 它会怼你、会反问你、会让你自己想清楚。

## 核心能力

- 🔍 **产品方案评审** — 用五层审视框架（灵魂之问 → 用户视角 → 设计原则 → 生态思考 → 长期主义）拆解你的产品方案
- ⚡ **需求排优先级** — 以用户价值为锚点，帮你砍掉不该做的需求
- 🆚 **竞品分析** — 从产品哲学层面解读竞争格局，而非简单的功能对比
- 💡 **产品直觉训练** — 通过持续对话，培养你"像张小龙一样思考"的能力

## 知识体系

Skill 内置了四份精心整理的知识文件：

| 文件 | 内容 | 用途 |

## 快速开始

### 方式一：CodeBuddy / WorkBuddy 用户

1. 将本仓库克隆到你的 Skill 目录：
   ```bash
   # 用户级（所有项目可用）
   git clone https://github.com/a18515373115-droid/AI-Allen-Zhang.git ~/.workbuddy/skills/AI-Allen-Zhang

   # 或项目级（仅当前项目）
   git clone https://github.com/a18515373115-droid/AI-Allen-Zhang.git .workbuddy/skills/AI-Allen-Zhang
   ```
2. 在对话中提到"产品设计"、"张小龙"、"需求评审"等关键词，Skill 会自动激活

### 方式二：手动使用

将 `SKILL.md` 的内容作为 System Prompt 喂给任何大模型（GPT-4、Claude 等），配合 `references/` 下的知识文件作为上下文，即可获得类似体验。

**AI 张小龙：**

> 
> 社交的本质是关系链。关系链不是功能，不是你加一个聊天框就有了的。它是用户花了十年时间沉淀下来的东西——他的家人、同事、那个不太联系但不能删的前同事。
>
> 抖音的基因是内容消费。内容消费和社交是两种完全不同的心智......

## 设计理念

这个 Skill 不是"张小龙语录复读机"。它的设计遵循几个原则：

1. **性格先于知识** — 先塑造说话方式和思维习惯，再灌入知识。一个只会背语录的 AI 没有灵魂
2. **框架可迁移** — 五层审视框架不依赖微信的具体案例，适用于任何产品
3. **边界清晰** — 明确声明是"模拟思维方式"，不编造语录，承认局限
4. **反教条** — 不把张小龙的话当圣旨。"我所说的都是错的"本身就是最核心的教义
